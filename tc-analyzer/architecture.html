RAG + Multi-Agent Pipeline for Terms & Conditions Risk Assessment

Input Layer
User Input
Input
📄 Document Loader
Accepts PDF (via pdfplumber) or raw text paste. Extracts clean UTF-8 text with page markers.
Preprocessing
Process
✂️ Semantic Chunker
RecursiveCharacterTextSplitter · chunk_size=1000 · overlap=200 · separators: paragraph → sentence → word
Indexing
Vector DB
🧠 Embeddings → FAISS Index
OpenAI text-embedding-3-small · 1536-dim vectors · FAISS flat L2 index · Top-K similarity search per agent query
Agent Layer (Multi-Agent)
🤖 Agent 1
Clause Extractor
RAG retrieval over 5 targeted queries. Identifies 8 clause types. Extracts verbatim excerpts. ReAct-style reasoning: Observe → Reason → Extract.
⚠️ Agent 2
Risk Analyzer
Receives extracted clauses + RAG context. Assigns low/medium/high risk level. Explains consumer impact in plain English.
📝 Agent 3
Summarizer
Synthesizes risk findings + document context. Produces TL;DR, rights, obligations, red flags, and overall verdict.
Scoring Layer
Risk Score
Output
📊 Weighted Risk Scorer
Score = Σ(wᵢ · rᵢ) / Σwᵢ where rᵢ ∈ {2, 5, 9} for low/medium/high. Data/Privacy weight=2.0 · Arbitration=1.8 · Liability/Auto-renewal=1.5
